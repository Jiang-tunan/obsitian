## 链接
#### Linux文件系统结构
```
磁盘上的数据
    ↓
inode（索引节点：存储文件属性、数据块指针）
    ↓
数据块（实际内容）
    ↑
目录项（文件名 → inode的映射）
```
**关键理解**：文件名只是指向inode的一个"指针"！

### 软链接 vs 硬链接 快速对比
|特性|软链接（符号链接）|硬链接|
|---|---|---|
|**本质**|快捷方式（指向路径）|另一个文件名（指向数据）|
|**命令**|`ln -s`|`ln`（不加-s）|
|**inode**|不同|相同|
|**跨文件系统**|✅ 可以|❌ 不可以|
|**链接目录**|✅ 可以|❌ 不可以|
|**原文件删除后**|❌ 失效（死链接）|✅ 仍然可用|
|**文件大小**|很小（存路径）|同原文件|
|**文件类型**|l（link）|-（普通文件）|
|**链接计数影响**|不增加原文件计数|增加原文件计数|
|**权限**|777（实际看原文件）|同原文件|

### 硬链接详解
#### 3.1 什么是硬链接？
**多个文件名指向同一个inode**，它们共享相同的数据块。

#### 3.2 命令语法
```bash
# 基本语法
ln 原文件 硬链接文件
# 示例
ln original.txt hardlink.txt
```
#### 3.3 查看方式

```bash
# 查看inode（-i选项）
ls -li
# 输出示例
12345 -rw-r--r-- 2 root root 27 Feb 15 10:30 original.txt
12345 -rw-r--r-- 2 root root 27 Feb 15 10:30 hardlink.txt
# ↑ 相同的inode号，链接计数=2
# 查看链接计数
ls -l
stat 文件名
```
#### 3.4 硬链接的特点
1. **同一个文件**：修改任意一个，另一个也变
2. **删除安全**：删除一个，另一个还在（链接计数减1）
3. **不能跨分区**：不同文件系统inode不通用
4. **不能链接目录**：防止循环引用
5. **只能文件**：不能给目录创建硬链接

#### 3.5 链接计数机制
```bash

# 每创建一个硬链接，计数+1
ln file1 file2   # 计数=2
ln file1 file3   # 计数=3
# 删除一个硬链接，计数-1
rm file2         # 计数=2
rm file1         # 计数=1（最后一个引用）
# 只有当计数=0时，文件才真正从磁盘删除
```
#### 3.6 目录的硬链接

每个目录都有2个特殊的硬链接：

```bash

ls -la /home/test/
drwxr-xr-x 2 root root 4096 Feb 15 10:30 .
drwxr-xr-x 5 root root 4096 Feb 15 10:30 ..
# .  指向目录本身（计数+1）
# .. 指向父目录（计数+1）
```
#### 3.7 为什么不能给目录创建硬链接？

```bash
ln /etc /etc2
ln: /etc: 不允许将硬链接指向目录
- 防止创建循环引用
- 防止目录树结构混乱
- 只能通过软链接链接目录
```

### 软链接详解

#### 4.1 什么是软链接？

**一个特殊的文件，内容是指向另一个文件的路径**。类似Windows的快捷方式。

#### 4.2 命令语法

```bash

# 基本语法
ln -s 原文件 软链接文件
# 示例
ln -s original.txt softlink.txt
```
#### 4.3 查看方式

```bash

# 查看
ls -li
# 输出示例
12345 -rw-r--r-- 1 root root 27 Feb 15 10:30 original.txt
12346 lrwxrwxrwx 1 root root 12 Feb 15 10:31 softlink.txt -> original.txt
# ↑ 不同的inode号，l开头表示链接，大小12字节（存路径名）

### 4.4 软链接的特点

1. **独立的文件**：有自己的inode
    
2. **存储路径**：文件内容是指向的路径
    
3. **可跨分区**：存的是路径，不是inode
    
4. **可链接目录**：常用
    
5. **原文件删除后失效**：变成"死链接"
    
6. **权限显示为777**：实际权限取决于原文件
```

### 完整实战演示

#### 5.1 准备环境

```bash

# 创建测试文件
echo "这是原文件内容，用于测试链接" > original.txt
cat original.txt
这是原文件内容，用于测试链接
# 查看初始状态
ls -l original.txt
-rw-r--r-- 1 root root 42 Feb 15 10:30 original.txt
#           ↑ 链接计数=1
```
#### 5.2 创建硬链接

```bash

# 创建硬链接
ln original.txt hard.txt
# 查看
ls -l original.txt hard.txt
-rw-r--r-- 2 root root 42 Feb 15 10:30 hard.txt
-rw-r--r-- 2 root root 42 Feb 15 10:30 original.txt
#           ↑ 链接计数变成2
# 验证inode相同
ls -i original.txt hard.txt
12345 original.txt
12345 hard.txt
# 修改硬链接
echo " >> 通过硬链接添加的内容" >> hard.txt
cat original.txt
这是原文件内容，用于测试链接 >> 通过硬链接添加的内容

# 原文件也跟着变了！
```
#### 5.3 创建软链接

```bash

# 创建软链接
ln -s original.txt soft.txt
# 查看
ls -l soft.txt
lrwxrwxrwx 1 root root 12 Feb 15 10:31 soft.txt -> original.txt
# 查看inode（不同）
ls -i original.txt soft.txt
12345 original.txt
12346 soft.txt
# 修改软链接
echo " >> 通过软链接添加的内容" >> soft.txt
cat original.txt
这是原文件内容，用于测试链接 >> 通过硬链接添加的内容 >> 通过软链接添加的内容
# 原文件也变了！
```
#### 5.4 删除原文件测试

```bash

# 删除原文件
rm original.txt
# 硬链接还在！数据仍然可访问
cat hard.txt
这是原文件内容，用于测试链接 >> 通过硬链接添加的内容 >> 通过软链接添加的内容
# 软链接失效了
cat soft.txt
cat: soft.txt: 没有那个文件或目录
# 查看软链接状态（死链接）
ls -l soft.txt
lrwxrwxrwx 1 root root 12 Feb 15 10:31 soft.txt -> original.txt
# 通常显示为红色或闪烁，表示"断链"
# 文件实际还在，因为硬链接计数还没到0
ls -l hard.txt
-rw-r--r-- 1 root root 72 Feb 15 10:32 hard.txt
# 链接计数变回1了
```
#### 5.5 清理


```bash

# 删除硬链接
rm hard.txt
# 删除软链接（软链接本身也需要删除）
rm soft.txt
# 现在文件才真正删除
```

### 链接计数变化全过程


```bash

# 初始状态
touch file.txt          # 计数=1
# 创建硬链接
ln file.txt link1.txt   # 计数=2
ln file.txt link2.txt   # 计数=3
# 查看计数
ls -l file.txt link1.txt link2.txt
-rw-r--r-- 3 root root 0 Feb 15 10:30 file.txt
-rw-r--r-- 3 root root 0 Feb 15 10:30 link1.txt
-rw-r--r-- 3 root root 0 Feb 15 10:30 link2.txt
# 删除原文件
rm file.txt             # 计数=2（link1和link2还在）
# 删除一个硬链接
rm link1.txt            # 计数=1（只剩link2）
# 删除最后一个硬链接
rm link2.txt            # 计数=0，文件真正删除
```

### 类比理解

#### 硬链接 = 同一个人的多个名字

- 张三（大名）、三儿（小名）指向同一个人
    
- 叫哪个名字都找到同一个人
    
- 人都活着，叫哪个名字都行
    
- 人死了（最后一个名字被遗忘），所有名字都没意义
    

#### 软链接 = 指向一个人的路标

- "张三的家往前走100米"（存的是路径）
    
- 路标本身是独立存在的
    
- 人搬家了（原文件移动/删除），路标指向错误
    
- 但路标本身还在，只是指向错了
